# Docker 教育訓練 — 講師教學手冊

> 總時長：約 2.5 小時（含休息）
> 對象：Docker 初學者（有基本 Linux/CLI 經驗）
> 形式：講解 + 現場 Demo + 學員互動

---

## 課前準備清單

- [ ] 確認講師電腦已安裝 Docker Desktop 並可正常執行
- [ ] 預先 pull 需要的 image（避免現場網路慢）：
  ```bash
  docker pull hello-world
  docker pull ubuntu
  docker pull nginx:alpine
  docker pull alpine:3.19
  docker pull node:20-alpine
  docker pull mysql:8.0
  docker pull python:3.12-slim
  docker pull redis:7-alpine
  docker pull wordpress:latest
  docker pull postgres:16-alpine
  ```
- [ ] 清理環境，確保 Demo 乾淨：
  ```bash
  docker rm -f $(docker ps -aq) 2>/dev/null
  docker volume prune -f
  docker network prune -f
  ```
- [ ] 準備好 Demo 用的檔案（見 Demo 腳本文件）
- [ ] 確認投影 / 螢幕分享設定，terminal 字體放大到學員看得清楚
- [ ] 如果學員要跟著做，提前通知安裝 Docker Desktop

## 各段落詳細教學指引

### 1. 開場（10 min）

**目標**：讓學員理解「為什麼需要 Docker」，產生學習動機。

**過場引言**：
> 「大家有沒有遇過一個狀況——在自己電腦上跑得好好的程式，部署到 server 就爆掉了？或是新人來的時候，光環境設定就花了一整天？今天我們要學的 Docker，就是來解決這些問題的。」

**講解重點**：
1. 傳統部署的痛點（用「在我電腦上可以跑啊」的梗開場）
2. Docker 的核心價值：**打包一次，到處執行**
3. 誰在用 Docker？（幾乎所有現代軟體團隊）
4. 今天的課程目標：結束後能自己寫 Dockerfile、用 Compose 管理多容器

**參考教材**：01-基礎概念.md — 「什麼是 Docker？」、「為什麼需要 Docker？」

---

### 2. 第一段：基礎概念（20 min）

**目標**：建立 Container 的心智模型。

**過場引言**：
> 「在動手之前，我們先花一點時間搞懂幾個核心觀念。這些觀念不只是面試會考，更是之後實作的基礎。」

**講解重點**：

**(a) VM vs Container（8 min）**
- 畫出架構對比圖（投影教材中的 ASCII 圖）
- 強調關鍵差異：Container 共享 Host kernel → 啟動快、體積小
- 類比：VM = 整棟房子，Container = 一間房間（共用水電）

**(b) Docker 架構（5 min）**
- Client-Server 模型：CLI → Docker Daemon
- 帶過核心組件：Client、Daemon、Image、Container、Registry

**(c) Image 與 Container 的關係（7 min）**
- 類比：Image = 蛋糕食譜，Container = 做出來的蛋糕
- 一個 Image 可以跑出很多 Container
- Container 停掉後，修改不會回寫到 Image

**參考教材**：01-基礎概念.md

### 5. 第二段：Dockerfile 與 Image 管理（20 min）

**過場引言**：
> 「剛才我們都是用別人做好的 Image。但實際工作中，你一定需要包自己的 Image。這就需要寫 Dockerfile。」

**講解重點**：

**(a) Dockerfile 基本結構（8 min）**
- 用一個 Node.js 範例講完整流程：FROM → WORKDIR → COPY → RUN → EXPOSE → CMD
- 每個指令用一句話解釋

**(b) CMD vs ENTRYPOINT（5 min）**
- 這是面試必考！用三個範例對比
- 記憶口訣：CMD 可以被覆蓋，ENTRYPOINT 不容易被覆蓋

**(c) Layer 與 Cache（4 min）**
- 畫出 Layer 堆疊圖
- 重點：把不常變的放前面 → 加速 build

**(d) Multi-stage Build（3 min）**
- 用 Go 的例子最震撼：800MB → 15MB
- 實務上前端 build 也很常用

**參考教材**：02-Dockerfile與Image管理.md

### 7. 第三段：Container 操作與資料管理（15 min）

**過場引言**：
> 「前半段我們學會了怎麼建 Image、跑 Container。但有一個問題——Container 被刪掉之後，裡面的資料就沒了。如果你跑的是資料庫呢？所以我們需要學資料持久化。」

**講解重點**：

**(a) 環境變數（3 min）**
- `-e` flag 和 `--env-file`
- 實務上用來傳資料庫密碼、環境設定

**(b) Port Mapping（3 min）**
- 畫出 Host:Container 的對應關係
- `-p 8080:80` 的意思

**(c) Volume 與 Bind Mount（9 min）** ← 這是重點
- Named Volume：Docker 管理，適合資料庫
- Bind Mount：你指定路徑，適合開發時同步程式碼
- 類比：Volume = 把 USB 隨身碟插進 Container

**參考教材**：03-Container操作與資料管理.md

### 10. 第四段：Docker Network（10 min）

**過場引言**：
> 「現在我們會跑單一 Container 了。但實際應用幾乎都是多個 Container 一起合作——web app 要連 database，對吧？所以我們得讓 Container 之間能互相通訊。」

**講解重點**：

**(a) 為什麼需要自訂 Network（5 min）**
- 預設 bridge 不支援 DNS → 只能用 IP 互連（不方便）
- 自訂 bridge 支援用 Container 名稱互連（推薦）

**(b) 實務做法（5 min）**
- `docker network create` → `--network` 指定
- Container 名稱 = hostname
- 帶過 host / none / overlay（知道有就好）

**參考教材**：04-Docker-Network.md

### 11. 第五段：Docker Compose（20 min）

**過場引言**：
> 「到目前為止，每次啟動多個 Container 都要打一堆指令。有沒有更方便的方法？有——就是 Docker Compose。一個 YAML 檔案搞定所有事。」

**講解重點**：

**(a) Compose 是什麼？為什麼需要？（3 min）**
- 對比：沒有 Compose 要打 N 個指令 vs `docker compose up -d` 一行搞定

**(b) docker-compose.yml 結構（10 min）**
- services / volumes / networks 三大區塊
- 逐一講解常用設定：image、build、ports、environment、volumes、depends_on
- `depends_on` + `healthcheck` 的搭配（面試常考）

**(c) 常用指令（3 min）**
- `up -d` / `down` / `ps` / `logs` / `exec`

**(d) 環境變數管理（4 min）**
- `.env` 檔案 + `${VAR}` 變數替換
- `env_file` 傳給 Container

**參考教材**：05-Docker-Compose.md

**面試重點快速回顧**（投影 06-實戰與進階.md 最後的表格）：

| 問題 | 一句話回答 |
|------|-----------|
| Container vs VM？ | Container 共享 kernel，更輕更快 |

| Volume vs Bind Mount？ | Volume 由 Docker 管理，Bind Mount 由你指定路徑 |

## 教學小技巧

1. **Demo 出錯不要慌** — 這反而是教學機會，帶學員一起看錯誤訊息、學會排錯
2. **隨時回顧** — 每講完一個段落，用一句話總結重點
3. **用類比** — Container = 便當盒、Image = 食譜、Volume = USB 隨身碟
4. **控制節奏** — 如果某段講太久，可以縮減 Demo 時間；反之則多做示範
5. **注意學員表情** — 如果看到困惑的表情，停下來問「到這邊有沒有問題？」
