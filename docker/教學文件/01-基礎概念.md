# 01 - Docker 基礎概念

## 什麼是 Docker？

Docker 是一個 **容器化平台（containerization platform）**，讓你能把應用程式和它所需要的一切（程式碼、執行環境、系統工具、函式庫）打包成一個標準化的單元，稱為 **Container（容器）**。

簡單來說：
> Docker 讓你的程式在任何地方都能用**相同的方式**運行。

---

## 為什麼需要 Docker？

### 傳統部署的痛點

```
開發者：「在我的電腦上可以跑啊！」
維運人員：「但在 server 上就是跑不起來。」
```

常見問題：
- 不同環境的套件版本不一致
- 作業系統差異導致程式行為不同
- 安裝步驟複雜且容易出錯
- 多個應用在同一台 server 上互相干擾

### Docker 的解決方案

把所有東西包進一個 Container，在任何有 Docker 的機器上都能執行：

```
開發環境 → 測試環境 → 正式環境
   ✅          ✅          ✅
  （完全相同的 Container）
```

## VM（虛擬機）vs Container（容器）

```
┌──────────────────────────┐    ┌──────────────────────────┐
│      Virtual Machine     │    │       Container          │
├──────────────────────────┤    ├──────────────────────────┤
│  App A  │  App B │ App C │    │  App A │  App B │ App C  │
│  Bins   │  Bins  │ Bins  │    │  Bins  │  Bins  │ Bins   │
│ Guest OS│Guest OS│GuestOS│    ├──────────────────────────┤
├──────────────────────────┤    │     Docker Engine        │
│       Hypervisor         │    ├──────────────────────────┤
├──────────────────────────┤    │       Host OS            │
│       Host OS            │    ├──────────────────────────┤
├──────────────────────────┤    │      Hardware            │
│      Hardware            │    └──────────────────────────┘
└──────────────────────────┘
```

| 比較項目 | VM | Container |
|---------|-----|-----------|
| 啟動速度 | 數分鐘 | 數秒 |
| 大小 | 數 GB | 數十 MB |
| 效能損耗 | 較大（需要完整 OS） | 極小（共享 Host kernel） |
| 隔離程度 | 完全隔離（獨立 OS） | 行程級隔離（共享 kernel） |
| 適用場景 | 需要不同 OS、高安全隔離 | 快速部署、微服務、CI/CD |

## Docker 架構

Docker 採用 **Client-Server** 架構：

```
┌─────────────┐     REST API     ┌──────────────────┐
│ Docker CLI  │ ───────────────► │  Docker Daemon    │
│ (Client)    │                  │  (dockerd)        │
└─────────────┘                  │                   │
                                 │  ┌─────────────┐  │
  docker build ──────────────►   │  │  Images     │  │
  docker pull  ──────────────►   │  │  Containers │  │
  docker run   ──────────────►   │  │  Networks   │  │
                                 │  │  Volumes    │  │
                                 │  └─────────────┘  │
                                 └──────────────────┘
```

### 核心組件

| 組件 | 說明 |
|------|------|
| **Docker Client** | 你輸入指令的地方（`docker` 命令） |
| **Docker Daemon** | 背景執行的服務，管理所有 Container |
| **Docker Image** | 唯讀的模板，用來建立 Container |
| **Docker Container** | Image 的執行實例 |
| **Docker Registry** | 存放 Image 的地方（如 Docker Hub） |

---

## Image 與 Container 的關係

用一個比喻來理解：

```
Image     = 蛋糕食譜（配方）    → 唯讀、可分享
Container = 根據食譜做出的蛋糕   → 可執行、可修改
```

- 一個 Image 可以建立多個 Container
- Container 是 Image 的執行實例
- Container 停止後，內部的變更不會影響原本的 Image

## 安裝 Docker Desktop（Windows）

### 前置需求
1. Windows 10/11 64-bit
2. 啟用 WSL 2（Windows Subsystem for Linux）

### 安裝步驟

```bash
# 步驟 1：啟用 WSL（以管理員身分開啟 PowerShell）
wsl --install

# 步驟 2：重新開機

# 步驟 3：下載並安裝 Docker Desktop
# 前往 https://www.docker.com/products/docker-desktop/
# 下載 Windows 版本並安裝

# 步驟 4：確認安裝成功
docker --version
docker compose version
```

## 基礎指令實作

### 1. 你的第一個 Container

```bash
# 執行 hello-world image
docker run hello-world
```

這個指令會：
1. 在本地找 `hello-world` image → 找不到
2. 從 Docker Hub 下載（pull）
3. 建立一個 Container
4. 執行 Container 並印出訊息
5. Container 自動停止

### 2. 進入 Container 內部

```bash
# 啟動一個 Ubuntu container 並進入 bash
docker run -it ubuntu bash

# 你現在在 Container 裡面了！試試：
cat /etc/os-release    # 查看系統資訊
ls /                   # 列出根目錄
apt update             # 更新套件清單
exit                   # 離開 Container
```

參數說明：
- `-i`（interactive）：保持 STDIN 開啟
- `-t`（tty）：分配一個 pseudo-terminal

### 3. 在背景執行 Container

```bash
# 在背景啟動 Nginx web server
docker run -d -p 8080:80 --name my-nginx nginx

# 開啟瀏覽器前往 http://localhost:8080
```

參數說明：
- `-d`（detach）：在背景執行
- `-p 8080:80`：把 host 的 8080 port 對應到 container 的 80 port
- `--name my-nginx`：給 Container 一個名稱

### 4. 查看與管理 Container

```bash
# 列出正在執行的 Container
docker ps

# 列出所有 Container（包含已停止的）
docker ps -a

# 停止 Container
docker stop my-nginx

# 啟動已停止的 Container
docker start my-nginx

# 移除 Container（必須先停止）
docker rm my-nginx

# 強制移除（不需要先停止）
docker rm -f my-nginx
```

### 5. 管理 Image

```bash
# 列出本地的 Image
docker images

# 從 Docker Hub 下載 Image
docker pull nginx
docker pull nginx:1.25    # 指定版本（tag）

# 移除 Image
docker rmi nginx
```

## 常用指令速查表

| 指令 | 說明 |
|------|------|
| `docker run <image>` | 建立並啟動 Container |
| `docker ps` | 列出執行中的 Container |
| `docker ps -a` | 列出所有 Container |
| `docker images` | 列出本地 Image |
| `docker pull <image>` | 下載 Image |
| `docker stop <container>` | 停止 Container |
| `docker start <container>` | 啟動 Container |
| `docker rm <container>` | 移除 Container |
| `docker rmi <image>` | 移除 Image |
| `docker logs <container>` | 查看 Container log |
| `docker exec -it <container> bash` | 進入執行中的 Container |

## 練習題

1. 執行一個 `nginx` container，讓它在背景跑，並把 port 映射到 `3000:80`
2. 用 `docker exec` 進入這個 nginx container，查看 `/usr/share/nginx/html/index.html` 的內容
3. 停止並移除這個 container
4. 下載 `alpine` image（一個超輕量的 Linux 發行版），進入後查看它的大小

# 練習 3

docker rm my-web

# 練習 4
